This simple example illustrates one way to create a Time Pattern for a standard work shift, create an exception to a resource, then associate our Time Pattern with that resource.

The resource in this example is a Resource named Worker. Please note that the Worker resource must be defined in Arena before these actions can be performed. Attach the Basic Process template, then double-click on the Resource module. In the spreadsheet interface, double-click to add a new row and create a Resource with the Name Worker and a Type of Based on Schedule. When using Calendar Schedules, the Schedule Rule for the Resource must be Preempt.

Creating the Time Pattern

Our standard work shift runs Monday through Friday, from the hours of 9:00 am to 12:00 pm, then from 1:00 pm (13:00) to 5:00 pm (17:00). Workers are Available when on duty. To create this time pattern:

  1. Open the Time Pattern Editor by choosing Edit > Calendar Schedules > Time Patterns... from the main menu.

  2. In the Time Patterns area on the left, right-click on the My Time Patterns folder and rename it Work Schedules.

  3. Right-click on the Work Schedules folder and choose New Time Pattern. The Time Pattern will be named New Time Pattern by default. Rename it Work Week Schedule.

  4. Because our work week spans only 5 days, and our daily shift will only span 7 hours out of 24, it is easiest to start with a state of Unavailable. Choose the following:

Time Pattern Type: Capacity
Default Value Type: State
Default Value: Unavailable
Duration (click on ellipsis): 1 Week
Scale Factor: 1

  1. Now we define the availability of our Worker resource using Time Spans.
    Right-click in the Time Spans area and choose New Time Span. The Time Span dialog will open.

  2. Choose the following:

Value Type: State
Value: Available
Start Day: Monday 9 hr, 0 min
End Day: Monday 12 hr, 0 min
Click OK.

Add Time Spans for Monday afternoon, Tuesday, Wednesday, Thursday, and Friday the same way. Note that military time must be used.

 

You will notice that the Lock Times checkbox is checked by default and cannot be changed. This is because this time pattern is one week in duration.

A bar graph representing the Time Pattern will appear in the lower right. You can navigate this bar graph using the button bar next to it. Right-click on the chart to change its properties.

Note that Time Patterns can also be viewed and defined via the Schedule Module in the Arena Basic Process panel. Time Patterns defined in this way will appear in the Time Pattern Editors, provided that the Format Type is specified as Calendar. However, Time Spans must always be defined using the Time Patterns Editor.

  1. Click OK.

Click here to see what the Time Pattern Editor will look like when you are done.

Creating an Exception

Our plant is closed on Labor Day, so our Workers will not be available that day. To create this Exception:

  1. Open the Exceptions Editor by choosing Edit > Calendar Schedules > Exceptions... from the main menu.

  2. In the Exceptions Editor, choose Exception > New... The Exception dialog will open.

  1. Enter or choose the following:
    Description:     Labor Day Holiday
    Exception Type:     Capacity
    Value Type:     State
    Value:     Unavailable
    Start Date:     Monday, September 1, 2003  0 hr, 0 min
    End Date:     Monday, September 1, 2003  24 hr, 0 min

  2. In the Elements area, click on the + sign next to the Resources folder. Choose the Worker resource, then click >  to add it to the Application area.  

  3. Click OK.  This exception is now tied to the Worker resource. This Exception will now appear in the Exceptions Editor. Click OK again.

Exceptions to resources do not display in Arena.

Click here to see what the Exception dialog will look like when you are done.
 

Associating a Time Pattern with a Resource

Now that we have defined a Time Pattern and an Exception, we can associate our Time Pattern with our Resource.

  1. Open the Composite Time Patterns Editor by choosing Edit > Calendar Schedules > Display Composite... from the main menu.

  2. In the Elements area, click on the + sign next to the Resources folder. Then click on the + sign next to the Worker resource. Choose Capacity.

Note that the Exception we defined earlier appears in the Exceptions area. This is because the Labor Day Holiday Exception is tied to the Worker resource. You can right click in this area to define additional Exceptions, or edit the existing Exception. Any Exceptions created here will appear in the Exceptions Editor.

  1. Choose the following:
    Value Type:     Time Pattern
    Value:     Work Week Schedule

  1. Click OK.

Click here to see what the Composite Time Patterns dialog will look like when you are done.

In Arena, you can tie your time pattern to a resource, and view or edit information defined in the Composite Time Patterns Editor by using the Resource module.
